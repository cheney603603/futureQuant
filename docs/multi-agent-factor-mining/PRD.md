# 多智能体因子挖掘与策略自动回测系统 - 需求文档 (PRD)

## 1. 项目概述

### 1.1 背景与目标

**背景：**
- futureQuant 框架已具备完善的回测引擎、因子计算和策略模块
- 当前因子挖掘依赖人工经验，效率低、覆盖面有限
- 缺乏系统性的因子质量保障机制（防未来函数、交叉验证等）

**目标：**
构建一个自动化的多智能体因子研究平台，实现：
1. **因子自动发现**：通过多 Agent 协作，从多维度自动挖掘有效因子
2. **质量保障**：内置防未来函数检测、时序交叉验证、样本权重优化
3. **策略自动回测**：将有效因子转化为策略并自动评估表现
4. **知识沉淀**：构建可复用的因子库，持续优化挖掘效率

### 1.2 目标用户

| 用户角色 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| 量化研究员 | 快速验证因子想法 | 高效挖掘、多维度评估 |
| 策略开发者 | 自动化因子研究与回测 | 流程自动化、结果可复现 |
| 投资经理 | 评估因子策略有效性 | 风险可控、收益稳定 |

### 1.3 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 因子挖掘效率 | 提升 5x | 相比人工挖掘，发现有效因子的时间缩短 5 倍 |
| 因子质量 | ICIR > 1.0 | 挖掘的因子年化 ICIR 不低于 1.0 |
| 回测覆盖 | 100% 防未来函数 | 所有通过验证的因子均无未来函数问题 |
| 系统稳定性 | 日志完整追踪 | 所有挖掘过程可追溯、可复现 |

## 2. 功能需求

### 2.1 多智能体因子挖掘模块

#### 功能描述
通过多个专业智能体协作，从技术面、基本面、宏观面等多维度自动挖掘有效因子。

#### 用户故事

**US-2.1.1 技术因子挖掘**
> 作为量化研究员，我希望系统能自动发现技术面有效因子，以便快速验证交易策略假设。

**验收标准：**
- [ ] 支持动量类因子挖掘（如 N 日收益率、动量加速度）
- [ ] 支持波动率类因子挖掘（如 ATR 比率、波动率变化率）
- [ ] 支持成交量类因子挖掘（如成交量异动、持仓量变化）
- [ ] 支持形态类因子挖掘（如突破、反转形态识别）
- [ ] 每类因子自动计算 IC、ICIR 等评估指标

**US-2.1.2 基本面因子挖掘**
> 作为量化研究员，我希望系统能挖掘基本面因子，以便捕捉供需关系驱动的交易机会。

**验收标准：**
- [ ] 支持基差因子挖掘（现货-期货价差、基差率）
- [ ] 支持库存因子挖掘（库存变化率、库存周期）
- [ ] 支持仓单因子挖掘（仓单注销率、仓单压力）
- [ ] 支持期限结构因子（展期收益率、期限结构斜率）
- [ ] 自动处理基本面数据的发布滞后问题

**US-2.1.3 宏观因子挖掘**
> 作为投资经理，我希望系统能挖掘宏观驱动因子，以便捕捉大类资产轮动机会。

**验收标准：**
- [ ] 支持宏观经济指标因子（GDP、CPI、PMI 等）
- [ ] 支持金融环境因子（利率、汇率、流动性）
- [ ] 支持商品指数因子（CRB、南华指数）
- [ ] 支持跨市场联动因子（股市-商品、债券-商品）

**US-2.1.4 因子融合与筛选**
> 作为策略开发者，我希望系统能自动筛选和融合多个因子，以便构建多因子策略。

**验收标准：**
- [ ] 支持因子相关性分析（剔除高相关因子）
- [ ] 支持因子有效性衰减分析（剔除时效性差的因子）
- [ ] 支持多因子加权合成（ICIR 加权、最大化夏普等）
- [ ] 支持因子正交化处理（减少共线性）
- [ ] 输出因子组合的综合评估报告

### 2.2 因子评估与验证模块

#### 功能描述
对挖掘的因子进行严格的质量验证，确保因子的有效性、稳定性和可交易性。

#### 用户故事

**US-2.2.1 防未来函数检测**
> 作为量化研究员，我希望系统自动检测未来函数，避免回测结果失真。

**验收标准：**
- [ ] 静态检测：分析因子计算代码，识别可能的未来函数（如使用未来数据的 shift 方向错误）
- [ ] 动态检测：在模拟回测中验证信号延迟执行后因子是否失效
- [ ] 数据时间戳校验：确保只使用 t 时刻及之前的数据计算 t 时刻因子
- [ ] 发布延迟处理：对于基本面数据（库存、仓单等），自动处理数据发布延迟
- [ ] 输出检测报告，明确标注存在问题的因子

**US-2.2.2 时序交叉验证**
> 作为策略开发者，我希望使用时序交叉验证评估因子稳定性，避免过拟合。

**验收标准：**
- [ ] 支持 Walk-Forward 交叉验证（滚动训练/测试窗口）
- [ ] 支持 Expanding Window 交叉验证（扩展训练窗口）
- [ ] 支持 Purged K-Fold 交叉验证（带清洗期的 K 折验证，避免数据泄露）
- [ ] 自动计算各验证窗口的 IC、ICIR、分层收益等指标
- [ ] 输出因子在不同市场环境下的表现对比

**US-2.2.3 样本权重计算**
> 作为投资经理，我希望对不同样本赋予合理权重，避免极端市场影响因子评估。

**验收标准：**
- [ ] 支持基于波动率的样本权重（高波动期降权）
- [ ] 支持基于流动性的样本权重（低流动性期降权）
- [ ] 支持基于市场状态的样本权重（牛市/熊市/震荡市分类）
- [ ] 支持自定义样本权重规则
- [ ] 输出加权后的因子评估指标

**US-2.2.4 多维度因子评分**
> 作为量化研究员，我希望从多个维度对因子进行综合评分，以便筛选最优因子。

**验收标准：**
- [ ] 预测能力维度：IC 均值、ICIR、IC 胜率、IC t 值
- [ ] 稳定性维度：IC 月度稳定性、年度稳定性、衰减速度
- [ ] 单调性维度：分层回测单调性、多空组合收益
- [ ] 交易成本维度：因子换手率、信号衰减速度
- [ ] 风险维度：最大回撤、下行波动率
- [ ] 输出综合评分排名及各维度详细报告

### 2.3 策略自动回测模块

#### 功能描述
将验证通过的因子自动转化为交易策略，并进行回测评估。

#### 用户故事

**US-2.3.1 因子策略生成**
> 作为策略开发者，我希望系统自动将因子转化为可交易的策略，减少手工编码工作。

**验收标准：**
- [ ] 支持单因子策略生成（因子值作为信号强度）
- [ ] 支持多因子策略生成（因子加权合成后生成信号）
- [ ] 支持信号阈值设置（因子值超过阈值才生成交易信号）
- [ ] 支持持仓周期设置（日频调仓、周频调仓）
- [ ] 自动生成策略代码（Python），可直接导入 backtest 模块运行

**US-2.3.2 风险控制规则**
> 作为投资经理，我希望策略内置风险控制规则，确保回测结果贴近实盘。

**验收标准：**
- [ ] 支持止损规则（固定比例止损、ATR 止损）
- [ ] 支持止盈规则（固定比例止盈、移动止盈）
- [ ] 支持仓位控制（波动率目标仓位、风险预算仓位）
- [ ] 支持集中度限制（单品种最大仓位、相关性限制）
- [ ] 支持回撤控制（最大回撤达到阈值降低仓位）

**US-2.3.3 回测评估报告**
> 作为量化研究员，我希望获得详细的回测评估报告，全面了解策略表现。

**验收标准：**
- [ ] 收益指标：总收益、年化收益、超额收益
- [ ] 风险指标：最大回撤、年化波动率、下行波动率
- [ ] 风险调整收益：夏普比率、索提诺比率、卡玛比率
- [ ] 交易统计：交易次数、胜率、盈亏比、平均持仓天数
- [ ] 分年度表现：各年度收益、风险指标对比
- [ ] 支持文本报告、HTML 报告、JSON 数据三种输出格式

### 2.4 因子库管理模块

#### 功能描述
持久化存储挖掘的因子，支持版本管理和性能追踪。

#### 用户故事

**US-2.4.1 因子持久化存储**
> 作为量化研究员，我希望因子能够持久化存储，避免重复计算。

**验收标准：**
- [ ] 支持因子元数据存储（名称、类别、参数、创建时间）
- [ ] 支持因子计算代码存储（可复现因子计算）
- [ ] 支持因子评估结果存储（IC、ICIR、分层收益等）
- [ ] 支持 Parquet 格式存储因子值（高效列存）
- [ ] 支持因子标签管理（有效/无效/观察中等状态）

**US-2.4.2 因子版本管理**
> 作为策略开发者，我希望对因子进行版本管理，追溯因子演进历史。

**验收标准：**
- [ ] 支持因子版本记录（参数调整、计算逻辑变更）
- [ ] 支持因子版本对比（不同版本的评估指标对比）
- [ ] 支持因子回滚（恢复到历史版本）
- [ ] 支持因子变更日志（记录变更原因和效果）

**US-2.4.3 因子性能追踪**
> 作为投资经理，我希望追踪因子在不同时期的表现，及时发现因子失效。

**验收标准：**
- [ ] 支持因子滚动 IC 追踪（按月/季度滚动计算 IC）
- [ ] 支持因子衰减预警（IC 连续下降触发预警）
- [ ] 支持因子失效标记（连续多个周期 IC 不显著自动标记失效）
- [ ] 支持因子表现归因（分析因子失效原因）

## 3. 非功能性需求

### 3.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 因子计算 | 单因子 < 1s | 单品种 5 年日频数据 |
| 因子评估 | 单因子 < 5s | 包含 IC、分层回测等完整评估 |
| 批量挖掘 | 100 因子 < 5min | 多品种多因子并行计算 |
| 回测执行 | 单策略 < 30s | 5 年日频数据事件驱动回测 |

### 3.2 可扩展性

- **因子扩展**：支持自定义因子类型，通过注册机制扩展
- **智能体扩展**：支持添加新的专业智能体
- **评估指标扩展**：支持自定义评估指标
- **策略模板扩展**：支持自定义策略模板

### 3.3 可维护性

- **日志追踪**：所有挖掘过程完整记录日志
- **异常处理**：异常因子自动标记，不影响整体流程
- **配置化**：核心参数支持配置文件管理
- **文档完善**：API 文档、使用示例、最佳实践

### 3.4 兼容性

- **Python 版本**：3.10+
- **依赖管理**：与现有 futureQuant requirements 兼容
- **数据格式**：复用现有 DataManager 数据格式
- **接口规范**：遵循现有 core.base 抽象类定义

## 4. 约束与假设

### 4.1 技术约束

- 基于现有 futureQuant 框架扩展，不进行大规模重构
- 日频数据为主，暂不支持高频因子挖掘
- 单机部署，暂不考虑分布式计算

### 4.2 数据约束

- 依赖现有 DataManager 提供的数据
- 基本面数据需考虑发布延迟
- 宏观数据通常月频/周频，需映射到日频

### 4.3 业务约束

- 因子挖掘为辅助工具，最终决策由人工完成
- 回测结果不作为实盘交易依据
- 需遵守交易所相关合规要求

## 6. 附录

### 6.1 术语表

| 术语 | 定义 |
|------|------|
| IC | Information Coefficient，因子值与未来收益率的秩相关系数 |
| ICIR | IC 信息比率，IC 均值 / IC 标准差 |
| 未来函数 | 使用未来数据计算当前信号，导致回测失真 |
| Walk-Forward | 滚动前向验证，模拟实盘部署场景 |
| 分层回测 | 按因子值将标的分层，评估各层收益单调性 |
| 因子衰减 | 因子预测能力随时间下降的现象 |
| Purged K-Fold | 带"清洗期"的 K 折交叉验证，避免数据泄露 |
